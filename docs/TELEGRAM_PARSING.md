## Техническая реализация

### Стек

- **Telethon** — чтение групповых чатов через userbot API
- **SQLite** — хранение дампов (`data/chats/messages.db`)
- **Категоризация v1:** regex + keyword matching (быстро, без AI)
- **Категоризация v2:** Claude Haiku через API (точнее, дороже)

### Схема БД

```sql
CREATE TABLE messages (
    id INTEGER PRIMARY KEY,
    chat_id INTEGER NOT NULL,
    message_id INTEGER NOT NULL,
    date TEXT NOT NULL,
    sender_id INTEGER,
    sender_name TEXT,
    text TEXT,
    reply_to INTEGER,
    media_type TEXT,
    category TEXT,         -- problem/solution/faq/contact/ad/info/other
    subcategory TEXT,      -- parking/skud/video/...
    processed INTEGER DEFAULT 0,
    UNIQUE(chat_id, message_id)
);

CREATE TABLE chats (
    id INTEGER PRIMARY KEY,
    chat_id INTEGER UNIQUE NOT NULL,
    title TEXT,
    username TEXT,
    members_count INTEGER,
    last_parsed TEXT,
    total_messages INTEGER
);

CREATE INDEX idx_messages_category ON messages(category);
CREATE INDEX idx_messages_date ON messages(date);
CREATE INDEX idx_messages_chat ON messages(chat_id);
```

### Переиспользуемый код

| Файл | Проект | Что берём |
|------|--------|----------|
| `services/telethon_reader.py` | ChatCompanion | Чтение групп, backfill в SQLite |
| `utils/channel_parser.py` | ChannelSpy Bot | Парсинг каналов, метрики |
| `scripts/auth_userbot.py` | TG bot posting | Авторизация Telethon |
| `config.py` | ChannelSpy Bot | Паттерн загрузки .env |

## Ограничения и риски

| Риск | Митигация |
|------|-----------|
| Telegram ToS — парсинг чатов | Личный аккаунт, не бот. Не массовый сбор. Данные не публикуются as-is |
| Персональные данные | Не сохранять ФИО, телефоны в публичный доступ. Анонимизация |
| Rate limits Telethon | Задержки между запросами, exponential backoff |
| Большой объём сообщений | Инкрементальный парсинг (по дате последнего сообщения) |
