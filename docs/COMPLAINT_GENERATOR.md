# Концепция генератора жалоб

> Автоматическая генерация юридически грамотных обращений в инстанции.

---

## Идея

Житель выбирает проблему → система генерирует готовые документы для ВСЕХ релевантных инстанций с подстановкой данных жителя и ссылок на законы.

**Ключевое отличие:** не один шаблон, а пачка документов — по каждой инстанции свой формат, свои ссылки на законы, свои реквизиты.

---

## Архитектура

```
                    ┌─────────────┐
                    │  Проблема   │  ← выбор из списка или описание
                    └──────┬──────┘
                           ↓
                    ┌─────────────┐
                    │  Маппинг    │  ← проблема → инстанции + статьи
                    └──────┬──────┘
                           ↓
              ┌────────────┼────────────┐
              ↓            ↓            ↓
        ┌──────────┐ ┌──────────┐ ┌──────────┐
        │ Шаблон   │ │ Шаблон   │ │ Шаблон   │
        │ УК       │ │ ГЖИ      │ │ Прокур.  │
        └────┬─────┘ └────┬─────┘ └────┬─────┘
             ↓            ↓            ↓
        ┌──────────┐ ┌──────────┐ ┌──────────┐
        │ + данные │ │ + данные │ │ + данные │
        │ жителя   │ │ жителя   │ │ жителя   │
        └────┬─────┘ └────┬─────┘ └────┬─────┘
             ↓            ↓            ↓
        ┌─────────────────────────────────────┐
        │  Пакет документов (PDF / текст)     │
        │  + инструкция куда отправлять       │
        └─────────────────────────────────────┘
```

## Реализация по фазам

### Фаза 2: Простой генератор (шаблоны)
- Jinja2 шаблоны для каждой инстанции
- Подстановка данных жителя
- Выход: текст (копировать) или PDF
- Интерфейс: на сайте (форма) или через бота

### Фаза 3: Умный генератор (AI)
- AI определяет категорию свободного описания проблемы
- Автоматический подбор инстанций и статей
- Генерация индивидуального текста (не шаблон)
- Отправка по email прямо из системы
