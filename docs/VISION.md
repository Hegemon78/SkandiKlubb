# SkandiKlubb — Видение проекта

> Полное описание идеи экосистемы для жителей ЖК Skandi Klubb.
> Дата создания: 2026-03-01

---

## Контекст

**Объект:** ЖК Skandi Klubb, Санкт-Петербург, Аптекарский проспект, 18, литера А
- 1222 квартиры, 31 подъезд, 4 очереди (корпуса)
- Застройщик: Bonava (NCC), 2016 год постройки
- УК: ООО «ЭКВИДА-СЕРВИС» (ИНН 7802355579)

**Исходная точка:** Markdown-база знаний `~/УК ОСС/` для подготовки Общего собрания собственников. Уже собрано:
- 9 файлов проблем (паркинг, СКУД, видео, коммуникации, ремонт, капремонт, безопасность, благоустройство, УК)
- 200+ юридически выверенных формулировок для повестки ОСС
- 3 шаблона писем в УК
- Проект ОСС на 13 вопросов
- Нормативная база (ЖК РФ, кворумы)

**Проблема:** Информация разрознена, жители не знают куда жаловаться, каждый пишет жалобы с нуля, нет единого источника правды о доме и проблемах.

**Решение:** Экосистема SkandiKlubb — сайт + Telegram-бот + автоматизация жалоб + парсинг чатов.

## B. Генератор жалоб (ключевая фича)

### Логика работы

```
Житель выбирает проблему
         ↓
Система подставляет данные жителя (ФИО, адрес, кв.)
         ↓
Для каждой инстанции — свой шаблон
         ↓
В шаблон вставляются ссылки на законы (ЖК РФ, КоАП)
         ↓
Одна проблема → пачка документов по ВСЕМ инстанциям
         ↓
Документ готов к отправке (PDF / текст / email)
```

### Инстанции для обращений

| Инстанция | Когда обращаться |
|-----------|-----------------|
| УК (ЭКВИДА-СЕРВИС) | Первая ступень — любая проблема |
| ГЖИ Санкт-Петербурга | УК не реагирует, нарушения содержания дома |
| Роспотребнадзор | Качество воды, санитария, нарушение прав потребителей |
| Прокуратура | Систематические нарушения, бездействие УК |
| Администрация МО | Благоустройство, территория |
| ГИС ЖКХ | Электронные обращения, фиксация в системе |
| Суд | Крайняя мера — возмещение ущерба |

### Маппинг: проблема → инстанции → статьи

| Проблема | Инстанции | Статьи закона |
|----------|-----------|---------------|
| Паркинг/кладовки затоплены | УК → ГЖИ → суд | ст. 161-162 ЖК, КоАП 7.22 |
| СКУД не работает | УК → ГЖИ | ст. 161 ЖК, Правила содержания ОИ |
| Камеры не работают | УК → ГЖИ → прокуратура | ст. 161 ЖК |
| Некачественная вода | УК → Роспотребнадзор → ГЖИ | СанПиН, ст. 7 ЗоЗПП |
| Капремонт — нецелевое использование | УК → ГЖИ → прокуратура | ст. 170-174 ЖК |
| Территория не убирается | УК → ГЖИ → администрация МО | Правила содержания ОИ |

## C. Telegram-бот (альтернативный интерфейс)

### Сценарий использования

```
Житель пишет боту о проблеме (текст или голос)
         ↓
Бот определяет категорию проблемы (AI или menu)
         ↓
Если данные жителя есть → сразу генерирует документ
Если нет → просит: ФИО, адрес, квартира, площадь
         ↓
Генерирует документ по всем инстанциям
         ↓
Отправляет: готовый документ + инструкция куда отправлять
```

### Команды бота

| Команда | Действие |
|---------|----------|
| `/start` | Регистрация, ввод данных жителя |
| `/complaint` | Генерация жалобы — выбор проблемы |
| `/status` | Статус текущих проблем дома |
| `/contacts` | Контакты УК и инстанций |
| `/faq` | Частые вопросы |
| `/oss` | Информация об ОСС |

### Технический стек

- Python 3.13+, aiogram 3.22+
- SQLite / PostgreSQL для данных жителей
- AI: Claude Haiku для категоризации проблем (опционально)
- Шаблоны: Jinja2 для генерации документов
- PDF: reportlab или weasyprint

### Переиспользуемый код

| Компонент | Источник | Что берём |
|-----------|----------|-----------|
| Архитектура бота | ChatCompanion | Структура handlers/, services/, FSM |
| Шаблоны документов | УК ОСС | 02_Шаблоны/* |
| База проблем | УК ОСС | 03_Проблемы/* |

## D. Парсинг Telegram-чатов (Telethon)

### Цели парсинга

1. **Проблемы** — собрать реальные проблемы жителей (частотность, описания)
2. **Решения** — какие решения уже предлагались/работают
3. **Контакты** — инстанции, мастера, которых рекомендуют жители
4. **FAQ** — частые вопросы (рекуператоры, вода, отопление)
5. **Реклама** — кто рекламирует в чатах (потенциальные клиенты для размещения на сайте)

### Источники (уточнить у пользователя)

- Основной чат жителей ЖК Skandi Klubb
- Инициативные группы (если есть)
- Связанные тематические чаты
- Чаты по очередям/корпусам (если есть)

### Категоризация сообщений

| Категория | Маркеры | Пример |
|-----------|---------|--------|
| Проблема | "не работает", "сломано", "когда починят" | "Ворота в паркинг опять не открываются" |
| Решение | "мне помогло", "я решил", "звонил в" | "Позвонил в ГЖИ, через неделю починили" |
| FAQ | частые вопросы, повторы | "Какой рекуператор ставите?" |
| Контакт | телефоны, email, ссылки | "Хороший сантехник: +7..." |
| Реклама | предложения услуг | "Делаю ремонт под ключ" |
| Информация | объявления от УК, новости | "Завтра отключат воду" |

### Технический стек

- Telethon (userbot API)
- SQLite для хранения дампов
- Категоризация: regex + keyword matching (v1), Claude Haiku (v2)

### Переиспользуемый код

| Компонент | Источник | Что берём |
|-----------|----------|-----------|
| Чтение групп | ChatCompanion/services/telethon_reader.py | Backfill истории, запись в SQLite |
| Парсинг каналов | ChannelSpy Bot/utils/channel_parser.py | Метрики постов |
| Авторизация | TG bot posting/scripts/auth_userbot.py | Telethon auth flow |
| Конфиг | ChannelSpy Bot/config.py | Паттерн загрузки .env |

## E. Монетизация (перспектива)

### 1. Реклама на сайте

- Анализ рекламы из чатов → предложить размещение на сайте
- Таргетированная аудитория: 1222 квартиры, платежеспособные жители Петроградского района
- Форматы: баннер, карточка мастера, спецпредложение

### 2. Доска мастеров

- Проверенные мастера от жителей (рейтинг, отзывы)
- Категории: сантехники, электрики, клининг, ремонт
- Платное размещение для мастеров

### 3. Генератор жалоб (freemium)

- Бесплатно: базовые шаблоны жалоб (текст)
- Платно: расширенные пакеты:
  - PDF с оформлением
  - Отправка по email в инстанции
  - Отслеживание статуса
  - Юридическая консультация (партнёрство с юристами)

### 4. Сезонные услуги

- Организация групповых заказов:
  - Мойка фасадов (скалолазы, весна)
  - Чистка вентиляции
  - Обслуживание рекуператоров
  - Промывка отопления
- Модель: комиссия с подрядчика или фиксированная оплата

### 5. Масштабирование

- Расширение на соседние ЖК (Петроградский район)
- White-label решение для других инициативных групп
- SaaS-модель: "экосистема для ЖК" — подписка для УК или ТСЖ

## Конкурентные преимущества

1. **Данные из первых рук** — парсинг реальных обсуждений жителей
2. **Юридическая грамотность** — шаблоны со ссылками на законы
3. **Автоматизация** — одна проблема → пачка жалоб по всем инстанциям
4. **Локальность** — заточен под конкретный дом и город (СПб)
5. **Существующая база** — УК ОСС уже содержит структурированные данные

## Риски и ограничения

| Риск | Митигация |
|------|-----------|
| Мало пользователей | Начать с активных жителей, показать ценность на конкретных кейсах |
| Юридические ограничения парсинга | Telethon — личный аккаунт, не массовый сбор. Не публиковать персданные |
| Устаревание информации | Регулярный парсинг + ручная модерация |
| Конфликт с УК | Всё в рамках закона, цель — конструктивное решение проблем |
| Сложность генератора жалоб | Итеративный подход: сначала простые шаблоны, потом AI |
