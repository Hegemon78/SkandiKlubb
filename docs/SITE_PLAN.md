## Стек сайта (определить при старте Фазы 1)

### Вариант A: Next.js (как Evolventa)
- **Плюсы:** знакомый стек, динамика, SSR, API routes, масштабируемость
- **Минусы:** тяжелее для старта, нужен хостинг с Node.js
- **Когда:** если планируется личный кабинет и генератор жалоб на сайте

### Вариант B: Статик-генератор (Astro / Hugo)
- **Плюсы:** быстрый старт, бесплатный хостинг (GitHub Pages, Cloudflare)
- **Минусы:** нет динамики без дополнительного бэкенда
- **Когда:** если основная интерактивность через Telegram-бота

### Вариант C: Гибрид — Astro + API на Python
- **Плюсы:** быстрый сайт + бэкенд для генератора жалоб
- **Минусы:** два стека
- **Когда:** если хочется лёгкий фронт + мощный бэкенд

**Рекомендация:** Начать с Next.js (Вариант A) — уже есть опыт (Evolventa), масштабируется под все фичи.
